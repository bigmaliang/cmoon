推送服务器主要接受三种类型的请求
	连接
	发送消息
	接受消息

使用load runner 通过以下方式对推送服务器进行测试:
服务器环境:
	Linux 2.6.18-92.el5
	Intel(R) Pentium(R) Dual  CPU  E2180  @ 2.00GHz
	4G 内存

1, 服务器每秒处理请求数
	模拟用户不停向服务器发起 发送 和 接受 消息的请求, 用以测试服务器每秒最大能够处理的请求数,
	load runner 在每秒向服务器端发起 1200 个请求时, 请求的响应时间在 0.5 秒以内.
	服务器端负载, cpu 占用最高 70%, 内存百兆左右(测试发送的数据量不大).

2, 在线人数
	模拟多个用户同时在线, 间隔一秒钟, 发起一次发送和接受消息的请求, 同时模拟了 1000 多个用户在线,
	请求能正常响应, 时间跟上面差不多.

以上测试分别以两种方式进行:
	1, TRUSTSEND, TRUSTCHECK, load runner 端不进行返回 session id 的处理, 此时, 一个上线
	用户最多在线 45 秒, 超时后, 服务器端会删掉该用户的相关信息, 之后的SEND 和 CHECK 会返回用户不在线的错误.

	2, SEND, CHECK, load runner 端进行 session id 处理, 使服务器端能够区分每个用户的请求,
	此时能够进行长时间测试, 不受超时限制.

	以上两种情况虽然工作方式不一样, 但对网络 和 服务器 的资源要求基本一致, 测试数据也基本一样
